# Code of Conduct

OpenEval should be a useful place to improve agent evaluation methods, compare harness behavior, and discuss proof boundaries without making contributors feel boxed out.

## Expected Behavior

- Be direct, kind, and specific.
- Critique ideas, code, and evidence rather than people.
- Keep discussions public-safe: do not post secrets, private transcripts, provider tokens, proprietary code, or local database content.
- Respect proof boundaries. If a result is inferred, partial, synthetic, local-only, or manually reviewed, say so plainly.
- Help keep issues reproducible with small fixtures, exact commands, and clear expected behavior.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing someone else’s private data, credentials, traces, or logs.
- Repeatedly derailing technical discussions after maintainers ask to refocus.
- Presenting fabricated evaluation results, fake verification output, or misleading benchmark claims.

## Enforcement

Maintainers may edit, hide, lock, or remove issues, comments, pull requests, or discussions that violate this code. Maintainers may also block users who repeatedly ignore these expectations.

For security-sensitive or private-data issues, use the reporting guidance in `SECURITY.md`.
